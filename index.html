Portafolio de Proyectos (QA)

TEST SUITE 1: Urban Scooter (Full Stack Testing)

Categoría: Pruebas Manuales / API / Mobile

1. Desafío

Como parte de mi práctica profesional integral, Urban Scooter necesitaba validar el flujo completo de renta y seguimiento de scooters. El reto principal era identificar defectos reales y comportamientos no documentados, verificando la sincronización crítica entre la Web, la App Móvil y la API. El objetivo era garantizar una experiencia de usuario estable y lógica.

2. Mi Rol y Herramientas

  • Rol: QA Tester (Ejecución E2E, Documentación).
  • Herramientas: JIRA, Postman, Chrome DevTools, Charles Proxy, TestRail.

3. Proceso

Abordé el proyecto descomponiendo los requisitos mediante mapas mentales para visualizar el comportamiento esperado.

  • Diseño de Pruebas: Apliqué partición de equivalencia y valores límite para optimizar la cobertura.
  • Ejecución Multiplataforma: Validé el formulario de renta en Web, probé estados offline y notificaciones en Mobile, y ejecuté pruebas de API para confirmar transiciones de estado HTTP.
  • Documentación: Registré cada hallazgo en JIRA con evidencia clara para los desarrolladores.

4. Resultados y Conclusiones

Este proyecto reveló defectos críticos, incluyendo la duplicación de pedidos y dependencias ocultas entre front y back-end. Entregué una documentación sólida que demostró inconsistencias reales, validando mi capacidad para gestionar un ciclo de QA completo con pensamiento crítico.

TEST SUITE 2: Urban Grocers – Web UI Automation

Categoría: Automation / Python / Selenium

1. Desafío )

Se requería validar de forma automática las reglas de negocio del campo "Nombre" en la creación de Kits de productos. La validación manual era frágil y propensa a falsos negativos debido a elementos dinámicos del DOM y comportamientos dependientes del timing.

2. Mi Rol y Herramientas

  • Rol: QA Automation Engineer.
  • Herramientas: Python, Pytest, Selenium WebDriver, Git/GitHub, VS Code.

3. Proceso

Para resolver esto, desarrollé un framework de automatización robusto:

  • Arquitectura: Implementé el patrón Page Object Model (POM) para hacer el código escalable y fácil de mantener.
  • Cobertura: Automaticé escenarios positivos, negativos, valores límite y manejo de caracteres especiales.
  • Estabilidad: Utilice WebDriverWait para sincronizar las pruebas con la carga real de la página, eliminando fallos por latencia (flaky tests).

4. Resultados y Conclusiones

Logré convertir un checklist manual tedioso en una suite de pruebas automática y confiable. Aprendí a manejar elementos dinámicos complejos y entregué un código limpio con un README técnico, listo para integrarse en un flujo de trabajo profesional.

TEST SUITE 3: Urban Grocers – API Automation

Categoría: Backend Testing / API

1. Desafío

El objetivo era validar el endpoint POST /api/v1/kits. Necesitábamos asegurar que el backend aplicara correctamente las reglas de negocio, rechazando datos inválidos y manejando errores de forma segura. Se sospechaba que el sistema aceptaba valores que debían ser bloqueados.

2. Mi Rol y Herramientas

  • Rol: API Tester / Automation.
  • Herramientas: Python, Pytest, Requests Library, Terminal.

3. Proceso

Realicé un análisis de "zonas grises" en la documentación para encontrar vulnerabilidades.

  • Automatización: Creé scripts que generaban usuarios y tokens de autenticación dinámicos para cada prueba.
  • Pruebas de Estrés de Datos: Envié peticiones con datos faltantes, tipos de datos incorrectos y longitudes extremas para ver cómo reaccionaba el servidor.
  • Estructura: Organicé el proyecto de forma modular (Helpers, Data, Tests) para facilitar su lectura.

4. Resultados y Conclusiones

Confirmé que el backend aceptaba valores fuera de lo permitido, lo que representaba un riesgo de integridad de datos. Entregué un framework que no solo prueba si "funciona", sino qué tan robusto es el sistema ante datos incorrectos.
